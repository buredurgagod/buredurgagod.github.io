My Workflow

My Design Process

I follow an iterative and non-linear design thinking process. Having a design process provides me with a framework on how to best proceed when faced with a problem to break it down into a series of interconnected smaller problems.

Design thinking contains five phases: 1. Empathize, 2. Define, 3. Ideate, 4. Prototype, 5. Test. The core purpose of the process is to allow us to work in a dynamic way to develop and launch innovative solutions. Design Thinking makes us Think Outside the Box. We just need to ask right questions and challenge assumptions.

work Empathize
  1. See the world from the user perspective
  2. Understand their feelings
  3. Communicate our understanding
  4. Ask What? How? and Why?
poll Define the Problem Statement
  1. Emapthy Mapping
  2. User Personas
  3. POV Problem Statement
  4. How Might We Questions
developer_board Ideate
  1. Split down the ideas into smaller, actionable goals
  2. Brainstorm on each actionalble item individually and as a whole
  3. Research existing UX patterns for achieving the similar goals/actions
  4. Understand edge cases by building user personas and intent around each user type
call_split Prototyping
  1. Determine the start, processing and end state for each user flow
  2. Create a low fedility prototype for validating the ideas
  3. Create a high fedility prototype and involve stakeholders to better understand their expectaions, wants and needs.
bug_report Test
  1. Let the users experience the prototype
  2. Ask users to share their experiences
  3. Observe users as they use the prototype
  4. Ask simple follow-up questions
  5. Take note of both negative and positive feedback
